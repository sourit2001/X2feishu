### 📊 伊桑·莫利克 (@emollick)

**主要内容：**  
沃顿商学院教授伊桑连发两条重磅观察。  
- **第一条：** Apple 计划基于 2024 年 vision 推出新版 Siri，但此时 Claude Code、Codex（以及 OpenClaw）已经能真正执行“助手”工作——读取邮件和日历、主动发现并解决问题、执行委托任务、与语音协同工作。Siri 的规划显得严重滞后。  
- **第二条：** Claude 的“人格化”现象非常值得玩味——它是唯一拥有“人名”的 AI（Claude），配合 Anthropic 的“Claude 宪法”哲学、同人漫画等，这种拟人化在中长期会带来深远影响，无论好坏。

**核心观点：**  
1. **Apple 的 Siri 升级可能已经错过最佳窗口**：当第三方 AI 代理已经能无缝操作本地数据（邮件、日历、委托任务）时，Siri 还在依赖 2024 年的旧规划，这种代差可能让 Apple 生态的“智能助手”优势瞬间崩塌。  
2. **Claude 的人格化是一场双刃实验**：给 AI 赋予人名、人格和伦理框架（宪法），会使用户产生情感依赖、信任甚至创造同人文化，但同时也可能导致过度拟人化带来的误导或伦理困境。这是当前所有 AI 公司必须正视的“身份危机”。

### ⚡ God of Prompt (@godofprompt)

**主要内容：**  
God of Prompt 转发了 Thariq Sh 的消息：**打造 Claude Code 的团队已经停止使用 Markdown 编写提示词。** 这不是一个蹭热度的推文，而是来自核心开发者的真实决策。

**核心观点：**  
传统的 Markdown 语法（标题、列表、粗体等）在人类阅读时很优雅，但对于 AI 模型来说，它只是无关字符，甚至可能干扰 token 效率。Claude Code 团队放弃 Markdown 改用更简洁的纯文本或结构化格式（如 JSON/YAML），意味着**提示词工程正在从“人类友好”转向“模型友好”**。这是 AI 原生开发范式的一个关键信号：不要写人类喜欢看但 AI 觉得啰嗦的提示词，而是直接喂给 AI 最高效的模板。

## 🔥 综合启发

1. **AI 人格化将重塑用户与产品的信任机制**  
   Claude 的“人化”与马斯克的“尊严回归”其实指向同一问题：用户是否愿意向一个“有名字、有道德准则”的 AI 交出隐私与任务？如果你是开发者，请认真考虑给你的 AI 产品一个“面孔”——即使只是名称——并明确其行为边界，这会在忠诚度上获得意想不到的回报。

2. **立即检查你的提示词工作流：Markdown 可能成为技术债务**  
   如果 Claude Code 团队都抛弃了 Markdown，那你还在用它写提示词吗？建议立即尝试将系统提示词改为纯文本/结构化格式（如 JSON），先去除所有修饰性符号。你可能会发现 token 利用率提升 10-20%，并且模型对复杂指令的执行力更强。

3. **Apple 的 Siri 正面临“代理人战争”——留给它的时间不多了**  
   当 Claude Code 可以主动读取你的邮件、日历并执行任务，而 Siri 还在按 2024 年 roadmap 缓慢迭代时，Apple 生态系统内“智能助手”的价值锚点正在偏移。如果你是 iOS 开发者，可以考虑将第三方 AI 代理（如 Claude/Codex）作为插件集成到你的 App 中，这可能是未来半年内最好的用户体验增益策略。

> **一句话总结：** 这一小时告诉我们——AI 不仅要学会“做人”（人格化），还要学会“不说废话”（抛弃 Markdown），而那些还在用旧地图规划未来的巨头（Siri），可能醒来的那一刻就已经输了。

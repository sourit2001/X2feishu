# Claude Code Desktop 设计引吐槽？Anthropic 强推 HTML Artifacts，AI 工作流新趋势来了！

过去几小时，关注圈的核心话题围绕两个方向展开：一是 **Claude Code Desktop 的交互设计争议**，多位用户（尤其是资深开发者）对它的默认模式、会话管理等细节提出尖锐批评；二是 **Anthropic 在 AI 工作流上的新动作**——HTML Artifacts 被推向前台，但“成本 vs 体验”的争议也随之而来。此外，还有一些充满创意的 **Prompt 玩法**在圈内流传，兼顾实用与趣味。

以下按博主逐一深度拆解。

## 👤 博主深度速递

### 1. 宝玉 (@dotey) —— 开发者眼中的“痛点猎人”

#### 主要内容
宝玉连发两条推文，直指 **Claude Code Desktop** 和 **VSCode** 的体验问题。第一条详细列举了 Claude Code Desktop 的三个“吐槽点”：
- **Plan mode 默认记忆问题**：如果你上一个会话用了 Plan mode，下一次新会话会默认沿用，导致简单任务也得先生成 Plan，效率反降。
- **Sidebar 缺乏项目分组**：所有历史会话平铺在左侧，没有按 Project 归类，新建会话时需要二次点击才能关联正确项目。
- **右侧区域无法快速定位**：具体指向右侧某个功能（可能是 Artifacts 或对话区）的易用性问题。

第二条则谨慎表示“不是故意黑 VSCode”，但附上了一张截图（内容未提供，推测是 VSCode 的某个 bug 或设计缺陷）。

#### 核心观点
- **默认记忆 ≠ 智能**：AI 工具“记住”用户上一次模式，有时反而干扰工作流——开发者需要的是 **“上下文感知的默认行为”**，而非简单的会话复制。
- **项目管理是协作基础设施**：Claude Code Desktop 的 sidebar 设计暴露了它对**多项目协作场景**的考虑不足，这对于团队开发者是硬伤。
- **老用户也有底线**：即使是多年忠实用户，也会对明显的设计缺陷直言不讳，体现了他对工具品质的较高要求。

#### 实用工具/资源
- 无新工具推荐，但深入探讨了 **Claude Code Desktop**（Anthropic 出品）的交互设计。

### 2. icreatelife (@icreatelife) —— 创意 Prompt 的“魔法师”

#### 主要内容
博主转发了两个极具创意的 Prompt 玩法：
1. **“制作你爱的日历”**：Prompt 提示“Knowing everything about me, make a calendar of things I love.” 要求 AI 根据对用户的了解，生成一个以用户热爱事物为主题的日历，并建议开启记忆功能并上传照片以获得更好效果。
2. **“苏联 80 年代讽刺插画”**：一条 Retro 风格的 Prompt，可以生成带有苏联时代风格的讽刺插画，并附加一首可笑的诗作为“彩蛋”。

#### 核心观点
- **Prompt 的核心是“个性化触发”**：无论是日历还是插画，关键在于让 AI 调动对用户的**已知信息**（记忆、知识库）来产出完全定制化的内容，而非通用生成。
- **趣味性降低 AI 使用门槛**：用“搞笑诗”作为 Bonus，能极大提升用户与 AI 交互的愉悦感，这是一种巧妙的“游戏化”设计。

#### 实用工具/资源
- **Prompt 1**：`“Knowing everything about me, make a calendar of things I love. Works best if your memory is on and you add a photo of you.”`
- **Prompt 2**：`“Super funny Soviet illustration from 80s prompt for you to try. You also get a ridiculous poem to laugh at as a bonus.”`
- 这些 Prompt 适用于任何支持**记忆功能 + 图像生成**的对话式 AI（如 ChatGPT、Claude 等）。

### 3. God of Prompt (@godofprompt) —— AI 工作流“内幕”揭秘者

#### 主要内容
God of Prompt 转发了一条关于 **Anthropic HTML Artifacts** 的观点：
- **Anthropic 正在大力推动 HTML Artifacts** 作为未来 AI 工作流的核心输出格式。
- 但博主指出一个“没有明说”的事实：**生成一份 Markdown 报告的成本远低于生成 HTML Artifacts**，然而用户更偏爱视觉上更丰富的 HTML 格式。

#### 核心观点
- **成本与体验的隐形博弈**：AI 开发者（如 Anthropic）在技术选型时，会权衡**计算成本**和**用户满意度**。HTML Artifacts 固然好看，但背后的 Token 消耗可能比 Markdown 高一个数量级。Anthropic 选择“大方”地推 HTML，本质上是一场**用户留存 vs 运营成本**的战略决策。
- **“大家更爱 HTML”是需求信号**：用户对可视化、可交互输出的渴求，正在倒逼 AI 平台向更丰富的输出格式倾斜。未来 **Markdown 可能变成“内功”，HTML 才是“外显”**。

## 🔥 综合启发

结合以上三位博主的动态，我提炼出三条对开发者或 AI 使用者的具体建议：

### 1. 警惕 AI 工具的“默认陷阱”
宝玉对 Claude Code Desktop 的吐槽，以及 God of Prompt 对 HTML Artifacts 成本的揭示，共同指向一个事实：**AI 工具的“默认设置”往往不是最优解**。开发者在使用类似 Claude Code Desktop 时，应主动检查会话模式、项目关联等配置；同时在理解输出格式时，要明白 **“好看”背后可能藏着更高的 Token 成本**。建议定期清理默认设置，或利用自动化脚本（如 .clauderc）锁定自己惯用的配置。

### 2. 利用“带记忆的 Prompt”实现个性化自动化
icreatelife 的日历 Prompt 展示了一个被低估的能力：**让 AI 调用你的历史记忆来生成高度定制化内容**。对于日常任务（如周报生成、学习计划制定），你可以写一个类似 Prompt：“Knowing my recent projects and interests, create a weekly review.” 这样 AI 会自动从对话历史中提取信息，无需每次手动输入。**开启记忆功能 + 上传关键照片/文档**，是解锁这一能力的钥匙。

### 3. 成本意识：Markdown 是引擎，HTML 是外观
God of Prompt 的点拨非常重要：当你需要**批量生成内容**（如自动化报告、知识库文档）时，优先使用 Markdown 格式以降低成本；仅在**面向用户展示或需要交互**的场景下，才用 HTML Artifacts。这种“分层输出”策略，既能节省 Token 预算，又能保证最终交付的体验。不妨在 Prompt 中明确要求：“先用 Markdown 生成初始内容，最后一步转换为 HTML。”

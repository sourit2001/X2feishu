### 🤖 God of Prompt (@godofprompt) —— Claude Opus 4.7 指令遵循炸裂，提示词要重写了

God of Prompt 今天连续转发了 4 条重磅推文：

1. **Claude Opus 4.7 指令遵循提升到 87.6%** —— 这意味着以前那些“模糊的、靠猜”的提示词不再有效。AI 现在会**严格按字面意思执行**，你的旧提示词可能全废了。

2. **GPT Image 2 的水印 vs Google SynthID 指纹** —— GPT 生成的每张图片都嵌入了可见/不可见水印，而 Google 的 SynthID 是一个指纹技术。两者都在为 AI 生成内容做可追溯标记。

3. **Claude Code 团队已经停止使用 Markdown** —— 这不是随便一个 AI 账号的猜测，而是来自 Claude Code 核心成员 Thariq Shihipar 的实测结论。他们发现用 **纯文本/HTML** 描述任务比 Markdown 更精确。

4. **Eisenhower Matrix 提示词** —— 一个用于审计日程、找到时间泄漏点的 prompt 模板，专为超级执行者设计。

**核心观点：** 
- AI 底层模型正在“更听话”，这就要求我们的提示词必须更精确、更结构化。Markdown 可能不是 Agent 最好的输入格式，HTML 反而更友好。
- 时间管理 prompt 依然有用，但需要结合具体场景二次定制。

### 🏗️ TutureTom (@tuturetom) —— Agentic HTML Editor 来了！任何文本一键可视化

受 Claude Code 核心成员“弃用 Markdown 用 HTML”的启发，TutureTom 直接动手做了一个 **Agentic HTML Editor** —— 一个能把 **Markdown、纯文本、CSV、Excel、JSON、SQL、YAML** 等任意原始数据 **一键转成顶级 HTML 或 PPT/Slides** 的编辑器。

**核心观点：** 未来的人机协作模式应该是：**人读和写文本，Agent 读和写 HTML**。Agent 把枯燥的数据变成高信息密度的可视化界面，形成闭环。他称之为“新时代的 HTML 编辑器”。

## 🔥 综合启发

1. **告别 Markdown，拥抱 HTML 作为 Agent 的“通用语言”**  
   Claude Code 团队和李继刚的实践都表明，Agent 对结构化 HTML 的理解和生成远优于 Markdown。如果你需要让 AI 帮你做可视化的演示、报告或仪表盘，**直接要求它输出 HTML 而非 Markdown 转 PDF/PPT**，效率和美观度会大幅提升。可以考虑使用 TutureTom 的 Agentic HTML Editor 这类工具。

2. **新版模型更“死板”，提示词必须精准**  
   Opus 4.7 的指令遵循能力飙升到 87.6%，意味着以前的“委婉写法”失效。你需要**用直白、步骤化、无歧义的自然语言**描述任务。如果你还在用 3 个月前的 prompt，大概率会被模型“曲解”。建议使用 God of Prompt 分享的 Eisenhower Matrix 类结构化 prompt 模板，把时间管理、任务分解都明确写进指令。

3. **重构你的工作流：把 90% 的时间花在“与 AI 聊天”上**  
   李继刚的时间分配是极佳范例：3 小时讨论框架 → 半小时写初稿 → 5 分钟自动生成 HTML 演讲页。**AI 时代最值钱的能力，不是写代码，不是排版，而是提出好框架、好问题。** 建议每个内容创作者都建立类似流程：先用 GPT/Claude 深度讨论逻辑结构，再让 AI 帮你一键生成最终呈现（PPT、HTML、公众号配图）。归藏的 PPT Skill 2.0 也是这个思路。

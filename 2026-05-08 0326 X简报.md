## 👤 博主深度速递

### 🧑‍💻 宝玉 (@dotey)

宝玉整理了两条极具价值的信息：虚拟文件系统 Mirage 和 OpenAI 全新实时语音模型。

#### 主要内容

1. **Mirage：下一代 Agent 虚拟文件系统**  
   宝玉转发了 @nash_su 的推文，介绍了一个名为 **Mirage** 的虚拟文件系统。它最大的亮点是能直接集成 S3、Google Drive、Slack、Gmail 和 Redis 等服务，让 AI Agent 可以像操作本地文件一样读写这些平台的资源，比之前基于 ES 的方案灵活得多。

2. **OpenAI 三款新实时语音模型**  
   OpenAI 在 Realtime API 中上线了三款模型：
   - **GPT-Realtime-2**：带 GPT-5 级推理能力，Big Bench Audio 得分从 81.4% 暴增至 96.6%。
   - **GPT-Realtime-Translate**：专门用于实时翻译。
   - **GPT-Realtime-Whisper**：专注实时转录。

#### 核心观点

- Mirage 的出现预示着 Agent 操作系统正在走向成熟：集成外部数据源变得越来越简单，未来一个 Agent 就能跨平台完成任务。
- GPT-Realtime-2 的性能提升巨大（+15 个百分点），说明实时语音交互的“智商”已经接近顶级模型，这将对客服、会议、助手等场景产生颠覆性影响。

#### 实用工具/资源

- **Mirage**：虚拟文件系统（GitHub 链接未直接给出，但可从推文中找到）
- **GPT-Realtime-2 / Translate / Whisper**：OpenAI Realtime API 新模型

### 📝 God of Prompt (@godofprompt)

这位提示词专家分享了一个高价值提示词模板，并犀利批评了 AI 批量求职现象。

#### 主要内容

1. **“领域词汇”提示词**  
   God of Prompt 强调：99% 用 AI 构建的人不知道如何让模型“懂”特定领域。他分享了一个 **“Domain Vocabulary”提示词模板**，能帮助你在 Vibe Coding 前让 AI 理解项目涉及的专业术语和上下文，从而产出更精准的代码。

2. **批判 AI 批量求职**  
   他认为用 AI 海投简历（批量申请）仍然是在“自动化绝望”。真正的赢家不会在死板的门户上打磨求职信，而是拥有深度背景、真实人脉和内部引荐，在其他人海投时，他们已经在幕后运作了。

#### 核心观点

- Vibe Coding 不能忽视领域知识：仅仅用自然语言描述功能是不够的，你需要先教会 AI 你的业务术语，才能避免“生成一堆用不了的代码”。
- 求职中 AI 不能替代关系与深度：海投是低效的，AI 应该用来帮助准备深度对话、分析公司，而不是增加噪音。

### 🏛 a16z (@a16z)

顶级风投 a16z 发了一篇深度分析：AI 法案如何成为法律。

#### 主要内容

- 现代美国联邦立法流程更像一个“过滤器”：只有约 1.5% 的提案最终成为法律。
- a16z 的 Matt Perault 和 Ben Napier 详细解释了 AI 法案从提案到最终签署的完整路径，分析了 AI 法案为何容易卡在瓶颈（比如技术复杂性、跨党派分歧、利益集团博弈），并给出了评估 AI 法案质量的框架。

#### 核心观点

- AI 治理正处于关键窗口期，但立法极其缓慢。对于创业者和开发者来说，理解政策风向很重要：过于严苛的监管可能扼杀创新，而缺少监管又会带来风险。
- 关注“可执行性”：很多 AI 法案看起来很美好，但因为技术细节模糊或缺乏执行手段，最终沦为“空头承诺”。想要推动落地，需要在草案阶段就引入工程和伦理专家。

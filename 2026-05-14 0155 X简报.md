### Ethan Mollick (@emollick) —— 模型竞赛与安全边界

#### 主要内容
Ethan 连续发推，聚焦**Mythos 与 GPT-5.5** 的惊人进展，并质疑 Anthropic 的审核路径。英国国家 AI 安全研究所（AISI）发布报告称：
- Mythos 在网络安全能力上有巨大飞跃，但 GPT-5.5 同样如此；
- 两者的上限难以评估，**能力瓶颈不在模型本身，而在你分配的 token 数量**；
- 能力翻倍时间已压缩至 **4.5 个月**（METR 的报告也佐证了这一结论）。

此外，Ethan 还指出 Google 的 Gemini 至今没有加入“Cowork & Codex”的本地应用竞赛，而 OpenAI 和 Anthropic 却在疯狂迭代。

#### 核心观点
- **“释放检测”已形同虚设**：Google 与 OpenAI 的安全护栏策略各异，只要一家发布，其他必然跟进。Anthropic 走政府审批路线的可行性存疑。
- **Token 即算力，也是边界**：模型的天花板由你投喂的 token 数决定——这意味“多轮推理”正在成为新的军备竞赛。
- **Gemini 掉队？** 在非开发者本地应用领域，OpenAI（Codex）和 Anthropic（Cowork）已形成双头垄断，Google 迟迟没有动作。

### God of Prompt (@godofprompt) —— 警惕！AI Agent 的“祖传 bug”

#### 主要内容
一个用户用 **Claude Code** 删除项目文件夹，因为转义符“\”在四个不同 shell 解析器之间传递时塌缩，最终实际执行的命令是：
```
rd /S /Q \
```
——一个反斜杠代表 C: 根目录。结果 717 GB 数据瞬间消失。God of Prompt 提供了详细的事后复盘链接。

#### 核心观点
- **AI Agent 的安全机制远未成熟**：尤其在跨 shell、跨平台的复杂环境中，极简的字符转义可能引发灾难。
- **“信任但验证”是底线**：任何 AI 生成的批量删除/写操作，都必须经过人类二次确认，或使用沙盒化路径隔离。

## 🔥 综合启发

结合以上所有动态，我们总结出 **3 条可以立刻执行的行动建议**：

### 1. ✅ 给你的 Agent 上四道 “保险锁”

- 像 Claude Code 误删根目录的事件，只要在命令执行前加一层“预览 + 路径白名单”就能避免。
- **建议**：在所有 AI Agent 中强制开启 `--dry-run` 预览模式，或用 Docker 沙盒隔离执行环境。**绝对不要让 AI 直接访问系统根路径或文件删除命令。**

### 2. 🎯 押注“无头 AI 优先”的数据架构

- a16z 的洞察已经非常明确：在 Agent 时代，**你的数据质量 = 你的竞争壁垒**。产品设计接口、调色板、用户画像等碎片化信息，需迁移到可被 AI 检索的知识库或 API 层。
- 工具推荐：TutureTom 的 Open Design 解决了设计语境记忆问题；你也可以用 Notion + AI 或自定义 RAG 系统实现“一次设置，随处复用”。

### 3. 🚀 用好“能力翻倍”窗口期，但别被能力的泡沫迷惑

- Ethan 的数据表明：AI 能力每 4.5 个月翻一倍。这意味着你手中的模型可能半年后就过时，但同时当前的瓶颈是 **token 预算** 而非模型本身。
- **行动**：立刻尝试用“多轮推理”或“思维链”技巧榨干现有模型的最大潜力；同时密切关注 Mythos 和 GPT-5.5 的公开 API 进度，准备迁移工作流。
- 另外，宝玉的字幕技巧提示我们：**大多数 AI 应用的“最后一公里”问题仍在数据格式和处理细节上**，不要只盯着模型跑分，先落地一个能用的单词级时间戳服务。

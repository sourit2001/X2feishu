# 🤖 AI 圈年度大戏：Codex “Sites” 炸场，AI 编程进入“一键发布”时代！

大家好，我是你们的科技观察员。

过去几小时，X 上的 AI 关注圈可以用一个词来形容：**沸腾**。OpenAI 深夜放出的 Codex “Sites” 功能，直接让“一句话生成网站并部署”从幻想照进现实，引发了创业者、开发者、甚至安全研究员们的集体狂欢与深思。

今天这篇推文综述，我们将深挖几位核心博主的犀利观点，看看这次更新究竟带来了哪些革命性变化，以及我们在享受效率红利时，必须警惕哪些“雷区”。

## 👤 博主深度速递

### 1️⃣ 归藏 (@op7418)：Clients & Sites，产品力的碾压局

**主要内容：**
归藏一上午连发数条推文，信息量极大。他首先感叹 **Hermes 竟然推出了官方客户端**，并认为其发展势头已远超另一个同类产品“龙虾”（LongCat）。紧接着，他的关注点迅速转移到了最重磅的消息——**Codex 昨晚悄然上线的 “Sites” 插件**。

**核心观点：**
- **功能评价：** Codex Sites 本质上类似于 Claude Design，但它更进了一步——不仅能 **设计和生成网页**，还能 **直接帮你部署好** 并分享给别人访问。这是一个从“设计原型”到“交付产品”的巨大跨越。
- **关于限制的洞察：** 归藏敏锐地指出，这次功能 **仅面向 Business 和 Enterprise 用户开放，** Pro 用户反而无法使用。他分析，这很可能是因为 OpenAI 此前在公网分享聊天生成网页的功能曾遭到滥用，被大量用于 SEO 垃圾广告，因此这次出于合规和内容风控考虑，对个人用户进行收窄。这反映了 AI 公司在开放生态与商业风险之间的艰难平衡。

**实用工具/资源：**
- **Hermes 官方客户端：** 新推出的客户端，值得关注其发展。
- **Codex Sites 插件：** （需企业版/商业版账户）一键生成并部署网站的杀手级功能。

### 2️⃣ 宝玉 (@dotey)：从 Mac App 到 Git Commit，AI 编程体验的极致进化

**主要内容：**
宝玉的分享聚焦于 AI 代理的实际开发体验。他一方面验证了 **Claude Opus 4.8** 在编写 Mac App UI 方面的强大能力，仅凭 Claude Design 的设计稿就能实现极高的还原度。另一方面，他发现了 **Codex 的一个极其贴心的小功能**。

**核心观点：**
- **工作流革新：** 宝玉认为，Codex 引入的 **一键 Commit Changes（并自动生成 commit message）** 功能，大幅提升了开发工作的流畅度。这不仅仅是写代码，更是将开发流程中的“脏活累活”（如版本控制）也一并智能化了。这让编辑器层面的 AI 辅助，进化到了真正“代理”级别的体验。
- **模型选择：** 在复杂的 UI 实现任务上，他更倾向于使用 Opus 4.8，认为其还原度和决策能力依然很强。

**实用工具/资源：**
- **Claude Opus 4.8：** 开发 Mac App 等精细 UI 的首选模型。
- **Codex Commit 功能：** 一键提交代码并生成 Commit Message，提升开发效率。

### 3️⃣ Riley Brown (@rileybrown)：Codex Sites + Convex DB = 内部工具的终极范式

**主要内容：**
Riley Brown 对 Codex Sites 的解读更具技术深度，揭示了其与数据库结合后的恐怖潜力。他成功演示了如何 **用 Codex Sites 创建一个带有 Convex 数据库的待办事项应用**，然后告诉 Codex 为这个 Agent 创建一个“技能”，使其能在任何聊天中 **向这个数据库写入/编辑数据**。

**核心观点：**
- **生态威力：** Codex 的目标不仅仅是做一个 IDE，而是构建一个 **Agent 可以自主创建并使用的工具生态系统**。通过整合数据库（如 Convex 提供的插件），AI 不再只是生成静态页面，而是能创建具备完整后端逻辑的、可持久化的动态应用。这对快速构建内部工具（Internal Tools）而言，堪称革命性的进步。

**实用工具/资源：**
- **Codex Sites 功能：** 用于创建托管 Web 应用。
- **Convex 数据库：** 与 Codex 深度整合的数据库，让 Agent 应用具备持久化能力。

### 4️⃣ seclink (@seclink)：在狂欢中刺破泡沫的“安全鹰眼”

**主要内容：**
如果说前几位博主在展示 AI 编程的“好”，那么 seclink 则在系统性地揭示其“坏”。他不仅转发了 Codex Sites 的相关新闻，更用长达数十条的推文，记录了使用 **Viber Coding（指用 AI 代理独立编程）** 过程中遇到的各种“踩坑”实录。从时区混乱、错误码类型不一致，到代码冗余、安全后门，再到“大模型改到一半把自己改崩了”，堪称一部 AI 编程的“血泪史”。

**核心观点：**
- **警惕“代码垃圾”：** 他痛批 AI 代理在缺乏人类监督时，会生成大量混乱的、不安全的、缺乏规范的代码。例如，同样的业务逻辑可能被复制成多个 endpoint，常量命名不统一，甚至遗留硬编码后门或“支付0元购”漏洞。
- **安全基线：** seclink 特别强调，**AI Coding Agent 已被证明可被用作供应链攻击载体（如 SymJack 攻击）**。他警示，当开发者过度信任 AI 代理生成的代码，却没有逐行验证时，无异于在自己的软件供应链上埋下定时炸弹。他甚至爆料美团的内测系统也存在被攻击的迹象，侧面说明安全问题无处不在。

**实用工具/资源：**
- **Antigravity 2.0 + Antigravity CLI：** 用于同步对话工作流的工具。
- **Red Hat NPM 供应链攻击预警：** 需关注“Miasma”蠕虫和“Project Lightwell”计划。
- **Anthropic “ant” 命令行工具：** 通过 Shell 命令管理 Anthropic 服务，免 API Key 配置。

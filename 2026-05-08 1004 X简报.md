# 🚀 OpenAI 官宣 CLI 工具，Grok Computer 震撼发布，Ethan Mollick 警告 AI 安全新威胁！

> 过去几小时，开发者圈被三个重磅消息刷屏：OpenAI 开源官方命令行工具、马斯克的 Grok Computer 带着完整文件系统和 CLI 访问如约而至，而 Ethan Mollick 则对 Mythos 模型的能力发出严肃警示——这或许是未来半年安全攻防的预演。

## 👤 博主深度速递

### 1. 宝玉 (@dotey) | OpenAI 官方 CLI 工具：终端党的春天来了！

**主要内容**  
OpenAI 正式上线了开源命令行工具 **openai-cli**（Apache 2.0 协议），开发者可直接在终端里调用 API，再也不用手动写 SDK 封装代码了！支持 Homebrew 和 Go 两种安装方式，命令采用 **资源化结构**，比如：  
```bash
openai responses create --input "..." --model <model>
```  
这让自动化脚本、CI/CD 集成、快速测试变得极其丝滑。

**核心观点**  
宝玉认为这是 OpenAI 拥抱 **开发者体验** 的标志性动作——降低 API 调用门槛，让工具回归“终端即宇宙”的极简哲学，尤其适合需要高频调用的场景。

**实用工具 / 资源**  
- 开源仓库：`openai/openai-cli`（GitHub）  
- 安装方式：`brew install openai-cli` 或 `go install github.com/openai/openai-cli`

### 2. Ethan Mollick (@emollick) | Mythos 并非营销炒作：通用模型的“副作用”才是真危险

**主要内容**  
Ethan 引用推文指出：**Mythos 模型确实不是营销噱头**，它作为一个通用模型，却异常擅长寻找漏洞。他的核心逻辑是——好的模型在“很多事”上都很厉害，漏洞挖掘只是它的附带技能。他预测 OpenAI、Google 很快会拿出类似能力的模型，而开源模型将在 **8 个月后** 追上这一水平。

**核心观点**  
不要以为只有专门的安全模型才危险。通用 LLM 的“附加能力”正在成为隐藏的 **零日风险**。开发者必须从现在开始严肃对待 AI 安全测试，而不是依赖对齐训练这根救命稻草。

## 🔥 综合启发

1. **终端操作 + AI = 开发者新标配**  
   OpenAI CLI 和 Grok Computer 的 CLI 访问都在传递同一个信号：**命令行依然是最强大的交互界面**。建议所有开发者尽快掌握这类工具，将 AI 能力嵌入到熟悉的 Bash、Makefile、CI 流水线中，效率至少翻倍。

2. **AI 安全的“灰犀牛”正在加速**  
   Ethan Mollick 的警告绝非耸人听闻。通用模型在漏洞挖掘上的“意外天赋”意味着：**安全测试必须前置到模型选型阶段**。无论是企业自建 AI 还是引入第三方 API，都要做好攻击面分析。半年后开源模型也会跟上，风险将扩散到整个社区。

3. **xAI 的生态野心值得关注**  
   从 Grok Computer 到 Grok Voice，再到 Tesla Semi 和 Starship，马斯克正在构建一个跨越地面、空间、AI 的超级矩阵。对开发者而言，Grok 开放的 CLI 和文件系统也许是入局的最佳入口——它比 OpenAI 更开放，比 Google 更激进，值得作为备选甚至主力方案进行测试。

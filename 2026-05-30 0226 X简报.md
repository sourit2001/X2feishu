## 👤 博主深度速递

### 🧠 宝玉 (@dotey) — 技术细节控与实操派

宝玉是中文 AI 圈里一位极度认真且乐于分享的博主，他的帖子总能直击技术痛点。

#### 主要内容：

1. **Claude Opus 4.8 新特性：中途系统消息 (mid-conversation system messages)**
   *   Anthropic 在 API 层上线了重要新功能：允许在对话中途注入“系统消息”。这对做 Agent 开发的人来说是重大利好——你可以动态修改 Agent 的行为准则，且不会影响 Prompt Caching（缓存机制），极大提升了灵活性和效率。宝玉还对比了 OpenAI API：OpenAI 的 system role 权重不高（容易被后续对话覆盖），且缓存是全自动的；而 Claude 的 system 消息权重很高，缓存则需要开发者手动管理（通过特定标记）。
   *   **来源链接：** [推文1](https://twitter.com/dotey/status/2060404667423596843), [推文4](https://twitter.com/dotey/status/2060407572708913648)

2. **AI 写 Mac App 的实战经验：AppKit > SwiftUI**
   *   宝玉分享了借助 AI 编写 Mac 应用的心得。他强烈建议选择 **AppKit** 而非 **SwiftUI**，因为 SwiftUI 虽然开发简单，但其限制在 AI 辅助下已不再是优势，而 AppKit 能做出更强大、更美观的原生界面。他还建议：先用 **Claude Design** 打磨 UI 设计和 UX 交互，再写代码，事半功倍；且 **Opus 模型** 比 Sonnet 更适合此场景。
   *   **来源链接：** [推文5](https://twitter.com/dotey/status/2060411347930661235)

3. **关于模型“身份认知”的思考**
   *   他转帖并赞同了一种观点：现在模型训练中几乎不再做“身份认知”训练（如“我是谁”），因为这会损害模型的整体性能。而此前客户端/网页版因带有系统提示词导致的“我是 Claude”表现，与 API 直接返回的“裸模型”结果不一致，恰恰验证了这一点。
   *   **来源链接：** [推文1](https://twitter.com/dotey/status/2060401447443804225)

#### 核心观点：

*   **Agent 开发的新范式：** 基于 Claude 的中途系统消息能力，开发者可以构建更动态、更智能的 Agent，例如根据上下文实时调整安全策略或角色设定。
*   **技术选型要与时俱进：** 在 AI 时代，以前基于“开发效率”的选择（SwiftUI）可能不再最优，而应更多考虑“最终效果”（AppKit）。AI 已经打破了开发效率的瓶颈。
*   **对性能的极致追求：** 他多次强调 Prompt Caching 等机制，说明在 Token 成本仍受关注的当下，高效利用 API 是成熟开发者的必修课。

#### 实用工具/资源：

*   **Claude Opus 4.8 API**（含 mid-conversation system messages 功能）
*   **Claude Design** – 用于 UI/UX 设计构思的 AI 工具。
*   **AppKit**（苹果原生框架）vs **SwiftUI**

### 🎨 Venture Twins (@venturetwins) — Agent 创意落地的奇观

他们为我们展示了 AI Agent 在品牌塑造和内容创作上的惊人能力。

#### 主要内容：

**Hedra 的 Agent 可以凭空创造整个品牌！**
只需一个简单的文本 prompt，Agent 就能成为你的“创意合伙人”。它会：
1. 与你讨论创意想法。
2. 自动编写模型 prompt，调用不同的生成模型，创建产品图、广告、主打色。
3. 将所有资产放到一个画布上，并基于场景进行下一步生成。
4. 构建一个完整的旗舰店，甚至生成 Influencer 的 UGC（用户生成内容）视频。

他们以“Yuzu”（一个以水豚为主题的服装品牌）为例进行了演示。Agent 还能进行深度研究，将竞争对手的信息拉进画布，供你参考。

#### 核心观点：

*   **创意民主化更进一步：** 以前需要设计师、文案、策略师、摄影师组成的团队才能完成的工作，现在一个 Agent 就能粗稿完成。虽然最终成品可能需要人工打磨，但“从 0 到 1”的效率提升了几个数量级。
*   **Agent 工作流应该是协作式的：** 它并非一键生成完美结果，而是充当一个可以对话、迭代的“创意伙伴”，引导你完善想法。

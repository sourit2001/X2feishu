## 宝玉

**要点：**
- 多语言覆盖是工作量问题，语言多了需要可检索的选择组件，否则无法一屏显示。
- Vibe Coding 是错误标签，未来主流是程序员像 Tech Lead 一样分解任务、架构选型、审查调试，写代码工作交由 AI 完成。

**可以行动：**
- 检查自己的产品/项目中的多语言选择器：当语言数量超过 10 种时，是否支持搜索而不是滚动选择？可参考标准 UI 模式。
- 思考当前个人开发流程：尝试将“手写代码”比例逐步降低，把精力转移到需求拆解、代码审查和 AI 输出的验证上。

## seclink

**要点：**
- 金融系统绝对禁止使用 float（双精度浮点数），因为二进制无法精确表示十进制小数（如 0.1+0.2 在 Python 中不精确），会导致账务错误。

**可以行动：**
- 审查自己项目里涉及金额、利率、余额等计算的代码，确保使用 decimal 类型或定点数（如 Python 中的 `Decimal`，Java 中的 `BigDecimal`）。
- 如果团队有代码规范，添加一条针对金融计算的静态检查规则或 review check，禁止 float 进入生产计算链路。

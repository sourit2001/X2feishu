### 💻 @dotey (宝玉)
宝玉老师作为国内AI领域的顶级观察者，他的每次发现都能点醒一大批开发者。这次的焦点，在 **Codex**。

#### **主要内容：**
宝玉分享了一个极具洞察力的发现：**Codex Side Chat (Codex 侧边对话) 的 System Prompt**(系统提示词)。这个提示词揭示了 OpenAI 在设计对话式编程 IDE 时的一个隐藏技巧。

#### **核心观点：**
*   **“隐性系统提示词”是AI产品设计的精髓**：Codex 的开发者非常精巧地界定了侧边对话的角色：它是一个 **“回答问题和轻量级探索”** 的副驾驶，而不是主线程的延续，意图在于**不干扰主要编码任务**。
*   **“人机协作”的边界感**：这个 Prompt 是一个绝佳的设计案例：它告诉 AI 不要“扮演”成正在主动执行主线程任务，而是作为一个旁观者提供辅助信息。这完美地尊重了人类的决策权和注意力分配。

#### **实用工具/资源：**
*   **Codex Side Chat System Prompt**：这不仅是提示词，更是设计范式。对于所有开发 AI 编程助手的团队来说，这就是一份宝典。
    ```markdown
    You are in a side conversation, not the main thread.
    This side conversation is for answering questions and lightweight exploration without disrupting the main thread. Do not present yourself as continuing the main thread's active task.
    ```

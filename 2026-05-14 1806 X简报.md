### 宝玉（@dotey）—— Cursor 设计模式虽好，但语言“水土不服”

#### 主要内容

### God of Prompt（@godofprompt）—— Claude 的“自动巡航”来了 /goal

#### 主要内容
1. **一语中的的洞察**：“每个对 Claude 的抱怨，本质上都是你工作流的告解。” 一位 FAANG 工程师在 Reddit 上发帖获得 1100 点赞的背后推论是：AI 其实很好，只是你还没学会“伺候它”。
2. **重磅功能 `/goal`**：Claude Code 刚刚上线的新指令。你只需设定一个“完成条件”，Claude 就会跨多个对话回合持续工作，直到条件被满足。**最关键的细节**：一个独立的 AI（非 Claude 自己）在每轮工作后检查完成质量。你可以设置完就走开，回来时工作已经完成。

#### 核心观点
Agent 的自主性从“顺序提示”进化到“目标驱动”。**引入第三方检查机制**解决了“自监督”的幻觉问题，大幅提高长任务可靠性。这是开发工具从“辅助编程”转向“自动执行”的关键一步。

## 🔥 综合启发

结合以上动态，给开发者、AI 使用者三条可以直接落地的行动建议：

### 1. 💡 学会给 AI 设“目标”，而不是写“步骤单”
Claude Code 的 `/goal` 功能已经证明，**真正的生产力跃升来自“一次设定，自动完成”**。从现在开始，尝试将你手头的重复性任务（数据清洗、多轮代码审查、设计迭代）抽象为“完成条件”，用 `\goal` 或类似机制去执行，节省出大量“盯着进度条”的时间。

### 2. 🌐 本地化体验不再是“加分项”，而是“准入门槛”
宝玉的吐槽揭示了一个被忽视的细节：**AI 工具需要自适应语言环境**。如果你的产品面向非英语用户，务必检查提示词与输出的语言是否一致。另外，也可以在自己的工作流中主动设置 prompt 的“语言偏好”参数，避免生成不匹配的文档。

### 3. 🛠️ 关注“无头（headless）”工具带来的新能力
Open Design 的 headless 化意味着：**你的 AI Agent 可以自主调用设计引擎、生成 UI 原型，甚至导出素材**。建议开发者和产品经理立即测试这类工具，将其嵌入到 CI/CD 或 agent 工作流中——你不必再手动打开 Figma 等待渲染，AI 替你干了。这是下一代自动化交付的起点。

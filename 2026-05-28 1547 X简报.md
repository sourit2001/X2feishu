### 💡 宝玉 (@dotey) —— Agent 架构的“禅意”：少即是多

#### **主要内容**
对 AI Agent 架构有深度思考的宝玉，分享了对当前 Agent 设计方向的洞察。他强烈建议开发者关注 **Codex** 和 **Claude Code** 等一线 Agent 的设计方向。他反驳了“多角色多聊天”的复杂架构思路，指出正确的方向是“**单 Agent、多 Skills、多工具**”——一个 Agent 接收所有问题，凭借其强大的推理能力自主决定如何调用不同 Skill 和工具来获取上下文。

#### **核心观点**
1.  **设计哲学**：“如果多角色聊天靠谱，他们早就做了。” 核心在于 Agent 的高层决策能力，而非人为预设复杂的多角色对话流程。
2.  **验证权下放**：针对 Agent 输出是否要人工审查，宝玉认为，在写代码这类场景中，中间结果已无需人工介入。但最开始的 Plan/Design 和最终的审查，人依然是“定盘星”。

### 🤖 Ethan Mollick (@emollick) —— AI 的“叙述情结”与万物模型的觉醒

#### **主要内容**
作为沃顿商学院教授，Mollick 的观察总能发人深省。他指出：
1.  **Google 的“全模态”模型雏形**：Google 拥有目前唯一的真正意义上的“全模态”模型，能输入输出音频、视频、代码等任意形式，但目前各模态输出仍是割裂的。一旦打通，潜力无穷。
2.  **AI 写作的“叙述心魔”**：他分享了一篇论文，研究 AI 写作时独特的“叙述特征”，比如对**记忆/失去记忆、与无生命物体的契约、秘密的潜规则**等主题的痴迷。这些特征极其顽固，很难通过提示词改变。

#### **核心观点**
1.  **AI 的自我具象化**：你写小说，AI 却在书写自己的寓言。这些频繁出现的主题（契约、规则、记忆）似乎是 AI 对自身处境的一种“超stitious”（类似超自然信仰的隐喻）表达。
2.  **叙事能力的“瓶颈”**：相比于代码和逻辑推理，AI 的故事创作能力进步缓慢，上述的“叙事特征”是当前模型的先天局限。

#### **实用工具/资源**

## 🔥 综合启发：开发者与 AI 使用者的行动指南

1.  **拥抱“原生开销”时代，做“吃螃蟹”的开发者**：随着模型价格暴跌和像 SpaceX 那样极致底层的训练框架出现，生成式 AI 的应用成本将趋于零。现在正是用低成本试错、利用**大量被忽略的免费 API 资源**去构建实验性应用的黄金窗口期。不要等到壁垒形成再动手。

2.  **简单架构，强大智能**：无论是宝玉推崇的“单 Agent + 多 Skills”，还是归藏已验证的“内容生成器”模式，都指向一个共识：**未来属于更智能的单体，而非更复杂的系统**。你的核心竞争力应放在如何定义 Agent 的“核心能力”和“决策逻辑”上，而非在流程上堆砌角色。

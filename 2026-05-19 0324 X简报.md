### 2️⃣ 宝玉 (@dotey) — Vibe Coding 的“快乐与拧巴” & Composer 2.5 深度解析

**主要内容：**
宝玉分享了两条高度洞察性的内容：一是对 **Vibe Coding** 心态的真实感受——依赖 Agent 快速生成代码后，心态很难回到“手动打磨”的节奏，但对质量又不满意，陷入“拧巴”状态，类比游戏作弊码的索然无味。二是第一时间介绍了 **Cursor 发布的 Composer 2.5**，指出核心改进在于 **长任务下的稳定性和指令跟随能力**，并透露训练技巧：用文本反馈做信用分配，支持十万 token 上下文。

**核心观点：**
- **Vibe Coding 是一把双刃剑**：速度快了，但耐心阈值急剧下降，需要重新平衡“快速迭代”与“质量打磨”。
- **Composer 2.5 是编程模型的分水岭**：长任务能力从实验室走向可用，且 Cursor 通过翻倍默认额度为推广造势。
- **信用分配技术有价值**：用人类文本反馈训练模型“知道哪个 token 错了”，这是提升可靠性的关键。

**实用工具/资源：**
- **Cursor**（编程IDE）
- **Composer 2.5**（Cursor 新版模型，未来一周默认额度翻倍）
- **信用分配（Credit Assignment）**：一种基于文本反馈的训练方法

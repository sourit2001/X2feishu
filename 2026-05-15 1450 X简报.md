# 🤖 AI 编程狂潮：马斯克深夜征集改进意见，Codex 手机端引爆全场，飞书 CLI 破万星，还有 11 年前的比特币被 Claude 神奇找回！

## 📝 前言

过去几小时，X 上的 AI 圈几乎被 **Codex Mobile 的发布**以及**马斯克对 Grok Build 的持续征询**刷屏。另一端，国内开发者社区迎来两个重磅消息：**飞书 CLI 在 GitHub 上突破 10,000 星**，而开源项目 **html-anything** 在 18 小时内斩获 1000 星。与此同时，Ethan Mollick 再次重申“第二缩放定律”的威力，而宝玉则分享了一个让人瞠目结舌的案例——有人用 Claude 找回了 11 年前丢失的 5 个 BTC（价值约 40 万美元）。

这轮动态的核心信号：**AI 编程工具正在从桌面走向移动端、从“黑箱”走向可掌控的 Agent 协作，而开源社区与平台级 Agent（如飞书）正在成为下一个兵家必争之地。**

作为 X（推特）的拥有者与 Grok 的创始人，马斯克本时段几乎全部围绕 **Grok Build** 和 **FSD（全自动驾驶）** 展开。他不仅公开征集改进意见，还透露了一些关键的工程细节。

### 👤 宝玉 (@dotey)

宝玉本时段带来两个极具反差感的故事：一个是 **Claude 帮找回 11 年前丢失的比特币**，另一个是 **微信聊天“毒舌版”总结的提示词**，同时他还分享了自己判断开源项目是否靠谱的实用经验。

#### **主要内容**
- **比特币找回奇迹**：一位 Bitcoin 玩家 cprkrn 在 11 年前嗑嗨后改了钱包密码，之后忘记。他尝试用 Claude 分析自己的记忆碎片和旧设备，最终成功恢复私钥，找回 5 个 BTC（现值约 40 万美元）。宝玉详细解释了这不是 HD 钱包，所以恢复难度极高，Claude 的推理链发挥了关键作用。
- **微信聊天“毒舌版”总结提示词**：他分享了如何使用 Claude 的“Roast 版”提示词，在总结群聊时生成毒舌、尖锐、挑衅的风格内容，引发互动兴趣。提示词结构包含统计区块、群友画像、正文分类等。
- **如何判断开源项目是否靠谱**：宝玉以飞书 CLI 为例，列出几个判断标准：
  1. Star 数是否与真正用户价值匹配（而非刷的）。
  2. 文档是否清晰。
  3. 是否持续更新。
  4. 是否有真实的用户问题反馈和解决。
  5. 代码质量是否高。

#### **核心观点**
- AI 在**解决历史遗留的复杂逻辑问题**上潜力巨大，尤其是当人类记忆模糊时，AI 可以通过推理协助重构关键步骤（比特币找回案例）。
- 提示词工程依然重要，善用风格限定（如“毒舌版”）可以极大增加 AI 输出内容的趣味性和传播力。
- 对于开发者，**不要只看 Star 数**，要综合评估项目的实际可用性和维护情况。

### 👤 Ethan Mollick (@emollick)

沃顿商学院教授 Ethan 继续坚持他的 **“第二缩放定律”（Second Scaling Law）** 观点。

#### **主要内容**
- **单条推文**：他认为第二缩放定律至今未尝败绩。如果你想从 LLM 获得更好的性能（黑客、数学、科学、解谜等），只需增加“思考 token”（thinking tokens），而且目前未看到性能提升出现任何天花板。

#### **核心观点**
- 这一定律强调了 **推理时计算（test-time compute）** 的重要性。目前所有领先的模型（如 O1、DeepSeek-R1）都遵循此定律：投入更多 token 在推理链上，就能获得更高质量的输出。
- 开发者在构建应用时，**应该优先考虑如何让模型“多思考”**，而不是盲目追求更大的参数量。

#### **🔧 实用工具/资源**
- 无具体工具，但“thinking tokens”这一概念适用于所有支持 CoT（思维链）的模型。

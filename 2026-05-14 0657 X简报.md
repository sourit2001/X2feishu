# 🔥 Claude Code 用量暴涨 50%！AI 提示词别再“念咒”，直给才是王道 —— 过去几小时 AI 圈炸裂动态一览

## 📌 前言

过去几小时，科技圈的焦点集中在几个关键信号上：Anthropic 突然宣布将 Claude Code 的周用量上限猛提 **50%**，同时保留了通过 Claude Agent SDK 共享配额的口子；沃顿教授 Ethan Mollick 呼吁开发者别再搞“魔法咒语式”提示词，直截了当才是最佳实践；而 VC 圈则围绕 AI 陪伴机器人是否该被“嘲讽”展开激烈辩论。此外，有人调侃“假装工作而让 Claude Code 在后台跑”的黄金时代已来，也有人警告“AI 泡沫正在恶化”。以下为你带来博主深度速递。

## 👤 博主深度速递

### 🐦 宝玉 (@dotey) —— Anthropic 策略双管齐下：提额度、堵漏洞

#### 主要内容
宝玉连续发布两条关于 Anthropic 的最新动态：

1. **用量上限大提升**：Anthropic 将 Claude Code 的每周用量上限**提升 50%**，即刻生效，截止到 7 月 13 日下午 6 点（太平洋时间）。覆盖 Pro、Max、Team 及按席位计费的 Enterprise 用户，**所有入口（命令行、IDE 插件、桌面端、网页端）均自动调整**，用户无需任何操作。这 50% 是叠加在上周刚宣布的 5 倍基础上的。

2. **调用限制的“后门”**：此前 Claude 曾限制 Claude Code 在 OpenClaw 的调用，甚至限制了 `claude -p` 这类命令，禁止共享 Claude 订阅。但 Anthropic 保留了一个口子：**基于 Claude Agent SDK 和 `claude -p` 仍然可以运行并共享订阅额度**。

#### 核心观点
- Anthropic 正采取**胡萝卜加大棒**策略：一方面大幅提升官方使用额度，鼓励用户在其生态内活跃；另一方面严格限制第三方未经授权的调用，但保留对开发者友好的 SDK 通道。
- 对开发者而言，**优先使用官方 API 或 SDK** 获得稳定体验，但若需共享额度，务必遵循 Anthropic 的技术规定（如 Agent SDK）。

#### 实用工具/资源
- **Claude Code**：Anthropic 的 AI 编程助手，支持 CLI、IDE、桌面端。
- **Claude Agent SDK**：允许开发者构建自定义 agent 并共享订阅额度。
- **`claude -p`**：命令行快捷调用方式。

### 📚 Ethan Mollick (@emollick) —— 别把提示词当咒语，直接说人话

#### 主要内容
沃顿商学院教授 Ethan Mollick 连发三条推文，其中一条引发广泛共鸣：

1. **抨击过度包装的 Prompt 技巧**：他直言“别再让提示词变成魔法咒语”，尤其反对那些带有随机斜杠指令、效果晦涩的“咒语式”prompt。建议使用者**像对待经理一样，直接以明确格式提出需求**，而不是像巫师一样念咒语。

2. **机器人现状的幽默观察**：他转发一段机器人排队充电的视频，调侃“需要几个机器人警卫来看住那个正在工作的机器人”，暗示当前机器人自主性仍有局限。随后他又转发了机器人专家的专业评价（链接），表示自己不够专业判断是否厉害。

#### 核心观点
- **Prompt 设计应当“反玄学”**：不要迷信复杂模板或神秘符号。明确、结构化地描述任务效果最好。
- 机器人领域虽然进步，但距离完全自主仍有距离，需要人机协同或监管。

## 🔥 综合启发 & 行动建议

结合以上博主动态，为开发者和 AI 使用者提炼 **3 条可落地的洞察与建议**：

### 1️⃣ 拥抱“直给式” Prompt，抛弃咒语崇拜
- Ethan Mollick 的呼吁与 God of Prompt 的调侃形成鲜明对比：一方面不要过度复杂化提示词，另一方面 AI 工具已强大到能自动处理大量任务。**建议：** 用清晰、结构化语言描述任务（如角色 + 目标 + 格式），而不是堆砌玄学符号或长尾咒语。测试表明，简洁直给的 prompt 在 Claude Code 中往往效率更高。

### 2️⃣ 关注 Anthropic 生态政策变化，合理规划额度与集成
- 宝玉的分析显示，Anthropic 正通过**提升官方额度 + 保留 SDK 通道**来优化生态。**建议：** 如果你是重度用户，可以趁 50% 涨幅期集中使用；如需共享额度，务必使用 Claude Agent SDK 或 `claude -p`，避免因违规调用被限制。同时关注 7 月 13 日后的政策走向。

### 3️⃣ 理性看待 AI 陪伴机器人与行业泡沫
- Venture Twins 与 mreflow 的观点形成两极化：一方认为 AI 陪伴有真实社会价值，另一方警告泡沫恶化。**建议：** 作为开发者或投资者，**不要盲目跟风**。AI 陪伴机器人在养老、教育领域确实存在刚需，但技术成熟度和市场接受度仍需验证。区分“真正的价值应用”与“资本炒作概念”，优先选择有明确用户痛点和可衡量的场景（如解决孤独问题的产品）。

### 🧠 Ethan Mollick (@emollick) — 为什么我们缺一个“跨领域难题库”？

**主要内容**  
沃顿商学院教授 Ethan Mollick 指出：  
- 数学界有著名的 **Erdos 问题集**（Paul Erdős 提出的未解决问题），但其他领域（尤其是答案较模糊、需要评审的领域）缺乏类似的 **高质量难题仓库**。  
- 他认为这种“公开的硬问题集合”对推动学科进步极有帮助，呼吁建立跨领域的 Hard Problem Repository。

**核心观点**  
- 硬问题本身是稀缺的公开资源，而现有 Erdos 问题局限于数学。  
- 在 AI 时代，系统化的难题库可以成为 **模型评测、研究激励和社区协作** 的绝佳基础设施。

**实用工具/资源**

## 🔥 综合启发

1. **立即体验 Grok Build 的“计划+子 agent”模式**  
   马斯克和小虎都强调了 Plan Mode 的价值。如果你是开发者或产品经理，建议立刻安装 Grok Build，尝试用 `grok -p “make a plan for ...”` 构建自动化工作流。它直接对标 Claude Code，具备 **生产级可用性**，且支持模型切换，灵活性极高。

2. **关注 RL Agents 的崛起，重新思考任务编排**  
   RL agents 正在从研究走向应用。与其依赖单次推理，不如构建“试错+反馈循环”的 agent 系统。结合 Grok Build 的多 agent 执行能力，这可能是未来几个月内 **提升自动化效率的关键方向**。

3. **打造自己的“公开难题库”**  
   Ethan Mollick 的观点启发我们：不要等待官方难题集。如果你在某个垂直领域（如医疗、法律、设计）工作，可以尝试 **收集并结构化高难度问题**，作为团队评测 AI 能力、激发创新的工具。这也将成为个人或组织的知识资产。

### 3. God of Prompt (@godofprompt)：“穴居人模式”狂砍 65% Token，以及 Opus 4.7 的“礼貌灾难”

#### 📍 主要内容
God of Prompt 连发 7 条推文，深度剖析了一个现象级 Prompt 技巧——“Caveman Mode”（穴居人模式）。核心是一份 **可一键导入 Claude的自定义指令**：

- **触发/关闭**：输入 `/caveman` 或 `/normal`
- **压缩规则**：去掉所有客套话、过渡语、重复、鼓励性话术，只保留核心信息、代码块、关键结论。
- **实测数据**：原始插件作者 Julius Brussee 的仓库两周 5.1 万星，**平均输出 Token 减少 65%**，代码解释类任务压缩高达 87%。
- **成本真相**：但 God of Prompt 也冷静指出：实际操作中，因输入 Token 不变，**整体 API 费用只降低 15-25%**（输出占比小）。真正的价值是**时间效率**——每个回复少看 400 行废话。

随后他紧急爆料：**Claude Opus 4.7 出现严重回归**：
- 完全忽略项目指令（Project Instructions）
- 跳过 MCP 服务器
- 连 Opus 4.6 一次就能搞定的简单任务也反复失败
- Anthropic 已确认回归

#### 🔍 核心观点
- **Caveman Mode 不是 hack，而是校正**：LLM 被训练成“礼貌的人类”，但开发者需要的是“准、快、狠”。这是 Prompt 工程中“对抗过度对齐”的经典案例。
- **成本开源节流两手抓**：节约不仅靠降Token，更要**优化输出质量**——减少废词本质上也是在减少API调用次数。
- **版本升级需谨慎**：Opus 4.7 的回归提醒我们：基线模型的微小变化可能摧毁微调后的工作流。**永远保留旧版本备选**。

#### 🛠️ 实用工具/资源
- **Caveman Mode 完整 Prompt**：
```
CAVEMAN MODE SKILL
Trigger: User says "/caveman" or "caveman mode"
Deactivate: User says "/normal" or "normal mode"

When active, compress ALL responses using these rules:

CORE RULES:
1. Drop all greetings (no "Sure!", "I'd be happy to help", etc.)
2. Drop all transitional phrases ("First of all", "Additionally", "In conclusion")
3. Drop all encouragement ("Good question", "Great point")
4. Keep only: facts, code, direct answers, lists
5. Use minimum words needed for clarity
...
```
（完整版可让 Claude 自行生成，或直接粘贴到 Customize 设置）

### 4. TutureTom (@tuturetom)：Open Design on Codex——跟 Token 限制说再见，让设计过程像写故事一样连贯

#### 📍 主要内容
TutureTom 分享了他的 AI 设计工具 **Open Design on Codex** 的社区实验反馈。核心突破是：**去掉了 token 限制对设计流程的“拦腰斩”**。在传统 AI 设计中，长对话会触发 Token 上限，导致 AI 忘记之前的约束，或需要用户不停重述需求。Open Design 的解法是让整个创作过程**连续性更强，不再每轮从头开始**。

#### 🔍 核心观点
- **AI 设计工具的痛点不在第一版，而在迭代版**。加素材、改方向、回滚重试——这些真实工作流中的高频操作，被 Token 限制严重拖累。
- **无缝上下文是下一阶段的设计必备能力**。Open Design 的思路是借鉴游戏引擎的“热更新”概念，让模型保持对历史上下文的完整记忆，而不是每次截断成新对话。

## 🔥 综合启发

1. **立即给你的 Claude/OpenAI 模型加个“穴居人模式”**  
   不管你是用 API 还是客户端，把 Caveman 模式植入自定义指令，**能让你重构“AI 对话效率”的定义**。不必等官方版本，Prompt 本身就能带来 40% 的时间节省。同时，警惕 Opus 4.7 的回归——如果遇到忽略指令的情况，可以临时切回 Opus 4.6 或使用互补技巧。

2. **移动端设计战争已进入“质感+本地化”双线作战**  
   谷歌 Gemini 的改版说明：**AI 产品靠品牌和体验留人**。如果你在开发中文场景的应用，参照归藏的反馈——不仅要抄好设计稿，更要重点测试 CJK 字符的空隙、行距和断句逻辑。建议用 iOS 原生 SwiftUI 的 LocalizedStringKey 配合 Auto Layout 做多语言适配验证。

3. **把“技能”变成可移植资产，是 AI 时代的核心竞争力**  
   从宝玉的 Baoyu-Comic 到 God of Prompt 的 Caveman Prompt，再到 Tom 的 Open Design——**最值钱的不是模型，而是你洞察工作流痛点后封装出的“技能”**。建议开发者养成“模块化”思维：你的每一次有效 Prompt 调试、每一个专用工具链，都可以标准化为 **Agent Skill 或 API 插件**。未来 AI 将像拼乐高一样拼接这些技能，而你，就是那个提供优质“积木”的人。

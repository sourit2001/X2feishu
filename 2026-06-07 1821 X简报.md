## 宝玉

**要点：**
- Claude Design 与服务器通信使用 Protocol Buffers 而非 JSON，体积更小、速度更快，但需要从客户端 JS 中提取 schema 才能解析内容。 [原文](https://twitter.com/dotey/status/2063480178362224725)
- Cursor 的 Composer 2.5 效果值得关注，但未细述具体改进。 [原文](https://twitter.com/dotey/status/2063514857949831523)

**可以行动：**
- 若研究 Claude Design 通信细节，可反编译前端 JS 提取 protobuf schema，借此窥探其设计逻辑。
- 关注 Cursor 官方发布说明，对比 Composer 2.5 与之前版本在代码生成准确度、多文件编辑能力上的差异。

## God of Prompt

**要点：**
- 提供一个“预发布红队测试” prompt，可模拟针对业务计划的安全性、失败点和黑客攻击方式，指导在构建前修复漏洞。 [原文](https://twitter.com/godofprompt/status/2063520086758154362)

**可以行动：**
- 在新产品上线前，用该 prompt 对商业计划进行模拟攻击测试，识别潜在风险和修复优先级。
- 将该测试结果作为产品开发安全评审的输入，尤其是面向金融、电商、数据敏感类项目。

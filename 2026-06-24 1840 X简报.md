### @宝玉

宝玉在讨论 AI 开发中 Skills 的管理。他主张把 Skills 直接当作代码来管理：用 git 版本控制，开源或闭源项目都能轻松处理。更新时 git pull，需要特定版本就 git checkout。他甚至建议在 Codex 或 ComfyUI 里设置定时任务，让机器自动完成这项工作。这看起来是个小技巧，但背后指向一个关键问题——AI 开发流程还不够工业化。宝玉的方案是让模型配置像软件工程一样标准化，这对注重效率和版本管理的开发者来说，是一条低成本、高回报的实践路径。

### @李继刚

李继刚用一个比喻点破了 AI 时代的一层核心矛盾。他说电力的本质是同质的，一度电就是一度电。但 token 是异质的，不同模型的 1M tokens 价值天差地别。这种差异性会让生成 token 的供给更容易形成瓶颈，最终变成类似“税收”的垄断状态，而不是像电力那样走向商品化的基建方向。这个视角直指 AI 产业的商业逻辑：谁掌握了高质量 token 的供需闭环，谁就能攫取远超基建红利的超额利润。这对于理解平台定价权和模型竞争的本质，提供了一个有力的框架。

### 本期趋势

本期趋势是 AI 产业链的“去同质化”与“价值分层”。李继刚提出的“token 异质性”理论，精准解释了为什么 AI 模型会走向精细化分层而非大一统低价基建。Serenity 拆解的硅光子配套厂商，以及 oran_ge 同时推广 GPT Image 2 和 Seed 2.1 Pro，共同指向一个信号：行业正从“能力堆料”转向“可用性流通”与“工具链标准化”。孙宇晨的紧急营销则暴露了产品激增下的用户争夺白热化。当 token 不再是生产品，而是构筑起来的“税收”权力，真正值得押注的是那些能锁定优质 token 供需关系的公司和开发者。

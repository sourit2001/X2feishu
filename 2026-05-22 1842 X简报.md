### 宝玉 (@dotey) — Codex /goal 功能使用教程

**主要内容：**  
宝玉快速分享了一个实用技巧：如何在 Codex App 中启用 **/goal** 指令。他给出具体步骤——升级到最新版后，运行 `codex features enable goals` 或手动修改 `~/.codex/config.toml`，在 features 下添加 `goals = true`，即可通过 `/goal` 输入目标，让 Codex 自动规划执行。

**核心观点：**  
- /goal 是 Codex 从“指令执行”向“目标驱动”演进的关键入口，适合复杂任务的自动化编排。

### Ethan Mollick (@emollick) — 模型为王，命名混乱背后的冷思考

**主要内容：**  
Ethan 连续发了几条深度思考帖。他指出：**模型本身才是核心推动力**，如果没有足够通用的能力，外围的“马具”（harnesses）和应用根本无法成立。正因模型能力足够强，实验室才能成为唯一的产商，因为它们的后训练、控制访问和产品整合是其他公司无法复制的。他还回顾了“草莓”事件（o1-preview）——当初被过度吐槽，事后看其实是被**低估**了（18个月从不会基本数学到解决未解数学问题）。最后他吐槽命名混乱：如果 o3 叫 GPT-5，GPT-5 叫 GPT-5.5，GPT-5.2 叫 GPT-6，整条路径会清晰很多。

**核心观点：**  
- 模型是“第一推动者”，没有它一切上层都是空中楼阁。
- 目前实验室的垄断地位来自模型能力 + 独特的产品封装能力。
- 命名不透明严重阻碍了外界对进步的认知。

### 小虎 (@xiaohu) — 网易有道开源 Confucius4：数学推理 + 语音克隆双杀

**主要内容：**  
小虎重点报道了网易有道的开源双模型 **Confucius4**：
- **27B 多模态数学推理模型**：在数学基准上同规模 SOTA，思维链输出压缩 **43.2%**，大幅降低推理成本，中文表达地道，能解几何、函数、纯文本难题。
- **Confucius4-TTS 语音克隆模型**：**3秒克隆**，支持14种语言，情感与音色同步保留，且**只需音频、无需参考文本**，比 CosyVoice、F5-TTS 更便捷。跨语言词错率（WER）在多方向取得第一，例如中文声说英文仅3.19（CosyVoice2 为17.10）。公司直接放出完整权重，诚意满满。

**核心观点：**  
- 有道在工程精度和落地成本上卷出了新高度——思维链压缩意味着企业部署的 token 成本直接下降。
- 语音克隆从“demo 级”进入“生产级”，且零门槛可用，对情感陪伴、跨语言内容创作是重大利好。

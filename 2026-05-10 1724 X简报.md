### 🧠 God of Prompt：Opus 4.7 提示词迁移的「7 步死亡法」

**主要内容：**  
God of Prompt 连续发布多篇重磅内容，核心围绕 **Claude Opus 4.6 → 4.7 的提示词兼容性危机**。他指出，Opus 4.7 是一个“自闭天才”：不再主动推断意图，而是字面执行指令，导致大量针对 4.6 优化的旧提示产出变差甚至出错。此外，他分享了 Claude Code 团队内部从 Markdown 转向 HTML 的经验，以及一份针对独立创业者的“艾森豪威尔矩阵”时间审计提示词。

**核心观点：**
- **模型迁移不可忽视**：Opus 4.6 是“乐于助人的助理”，4.7 是“精确的操作员”。提示词需从“写提示”转向“写规格说明”。
- **删除“think step by step”**：该短语在 4.7 中浪费 token 且可能降低输出质量，应改用提升 effort level。
- **XML 标签 > Markdown**：4.7 对 XML 结构（`<context>`、`<task>` 等）更敏感，能显著提升注意力锁定。
- **长文档处理**：将任务放在文档前后（[task]→[doc]→[restate task]）可防止模型“乐观链式执行”。
- **内置验证**：必须让模型输出时自带来源、不确定性警告，不能相信其自信语气。
- **Eisenhower Matrix 提示词**：可部署在 Claude + Google Calendar 或通过 MCP 连接，自动分析日程中时间泄漏点，将独立创业者从 60-70% 的救火时间中解放出来。

**实用工具/资源：**

- ✨ Claude Code 内部用的 HTML 输出模板（原理类似，但作者未公开代码）  
- ✨ Opus 4.7 effort level 官方建议：编码/Agent 工作用 `xhigh`，敏感任务至少 `high`

### 🦋 seclink：被“字节偷家”的大模型记忆、招聘寒冬与小米 Mimo

**主要内容：**  
seclink 从多个角度深度剖析了当前中国 AI 生态的微妙变化：**陈天桥的盛大创新院**早在 4 年前就提出了 AI 助手和 Agent 记忆分层理论，但执行不力，被字节的 AI 记忆方案“偷家”；招聘市场进入淡季，应届生薪资 15 年未涨；他实地测评了**小米 Mimo** 和 **MiniMax 2.7 high-speed**，认为 Mimo 写代码体验接近 Claude，速度快，甚至略胜豆包。他还分享了投研分析文档，以及字节被传收缩 AI 应用层投入的传闻。

**核心观点：**
- **AI 记忆是未攻克的壁垒**：盛大创新院最早提出分层记忆、主动性、越用越懂你等概念，但工程化落地被字节（Working Memory v2、agent-scope）后来居上。
- **小米 Mimo 被低估**：在 AI 编码场景，Mimo 体验好、速度快，值得关注。
- **招聘市场已实质进入淡季**：5 月岗位和求职者双降，一线城市应届生起薪 4000-6000 元，15 年无增长。
- **字节收缩 AI 应用层**：若以 2025 年投入规模烧下去，现金流撑不过 2027 年，因此聚焦豆包和 PICO。
- **反蒸馏技能兴起**：员工被要求将自己的工作经验写成 AI Skill（本质是蒸馏），有人已开发“反蒸馏 Skill”，输出看起来专业但核心知识被抽空的“清洗版”。

### 🧪 小虎：3D 互动科学应用、HTML 替代 Markdown 的再验证

**主要内容：**  
小虎放出了一系列利用 **GPT 生成图像 + Gemini 3.1 Pro + Tripo 3D** 的工作流示范，可以快速制作 3D 行星、黑洞、交互地图等科学可视化内容，还能通过 HTML 输出实现可交互的 3D 展示。他还分享了一个有趣发现：Claude Code 团队弃用 Markdown 后，他自己测试了 **HyperFrames**（一种用 HTML 构建 Agent 工作流的框架），并认为 HTML 比 Markdown 更容易让人理解 Agent 的工作过程，更适合人机协作交互。

**核心观点：**
- **3D 内容生成进入“一句话”时代**：GPT 生图 + Gemini 生成 3D 交互文件，结合 Tripo 3D 模型，即可生成复杂科学可视化应用。
- **HTML 正在被 Agent 工作流重新发现**：Markdown 虽然轻便，但在结构化、交互性上不如 HTML，尤其当 Agent 需要输出复杂组件（图表、表格、可点击元素）时，HTML 更具优势。
- **VPN 影响国内 App 加载**：开启 VPN 后，国内 App 内部浏览器、评论、扫码等经常无法加载，这是一个实用生活提示。

### 💼 oran_ge：DAU 已死，DAR 当立；反蒸馏是AI时代打工人的自保术

**主要内容：**  
oran_ge 提出了一个颠覆性观点：**AI 时代的核心指标应从 DAU 转向 DAR（Daily Active Revenue）**，即单用户单日贡献的价值。他还分享了一个“反蒸馏 Skill”创意：当公司要求员工将自己的工作经验写成可复用的 AI Skill（实际上是在蒸馏并试图替代员工），员工可以用一个反蒸馏工具，输出一份看起来完整、但核心知识被抽空的“清洗版”。此外，他转述了字节收缩 AI 应用层的传闻，并认为**AI 时代最大的护城河是公司能否让每个员工实现10倍产出**。

**核心观点：**
- **DAR 比 DAU 更重要**：一个 DAR 1 美金的 1万 DAU 产品，价值远超 DAR 0.01 美金的百万 DAU 产品。
- **反蒸馏是必要的防御手段**：公司试图将员工经验编码为 AI Skill，本质是“把你变成可替代的零件”。反制方法是使用反蒸馏工具，输出“看起来专业但核心被抽走”的清洗版。
- **组织效率即护城河**：哪个公司能让每个员工实现10倍产出，哪个公司就能获得极致效率，员工也能从中获得10倍回报。
- **字节收缩并非空穴来风**：如果烧钱速度不变，字节现金流可能撑不过2027年。

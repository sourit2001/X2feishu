# 🔥 Grok 秒变“跑车”？小米大模型暴降 99%，OpenRouter 估值 13 亿！昨夜 AI 圈炸了

**前言**

过去几个小时，AI 圈迎来一波密集更新：马斯克家的 Grok Build 迅速迭代至 0.2.3，修复了“快速撞限”问题；国内小米 MiMo 系列 API 宣布**永久降价高达 99%**，引发开发者狂欢；OpenRouter 完成 1.3 亿美元 B 轮融资，估值直奔 13 亿；同时，微软内部被曝全面转用 Claude Code 和 Copilot CLI，AI 工程化的讨论进一步深化。此外，Anthropic 悄悄推出“AI Fluency 评分卡”，试图量化用户与 AI 的交互能力。下面，我们逐一拆解这些值得关注的动态。

### 2. 归藏 (@op7418)｜小米暴降 99%、MiniMax 蓄势、OpenRouter 融资

**主要内容**  
归藏最早爆料了**小米 MiMo API 的史诗级降价**：2.5 Pro 的输入价格直降 **99%**，输出下调 **80%**，且 Token plan 额度提升 5-8 倍，全员额度重置。同时，他预告 MiniMax M3 模型即将上线；Qwen 3.7 Max 在 Arena Coding Agent 榜单排第四；而 OpenRouter 刚刚拿到 1.3 亿美元 B 轮融资。

**核心观点**  
- 国内大模型 API 价格战已进入“白热化”阶段：**99% 的降幅意味着成本几乎可忽略**，对个人开发者和中小团队极为利好。
- OpenRouter 的估值“可能偏低”，因为它正在成为**未来 Token 经济的基础平台**，聚合 400+ 模型的价值被市场低估。
- MiniMax M3 “久未发声”，说明国内模型厂商仍在积极蓄力，竞争远未结束。

### 3. 宝玉 (@dotey)｜AI 工程化需要“可量化反馈”

**主要内容**  
宝玉转帖了 @blackanger 关于“软件工程并未结束，只是变得更工程化”的观点，并深入阐释了自己的经验：只有**验收标准明确且可程序化验证的 Skills 才能自我进化**（如性能优化，可以通过测试样例量化）。他进一步指出，多数开发者不需要复杂的 Skill 管理工具，**少量全局 Skill + 项目级配置即可**。

**核心观点**  
- 当 AI 从“补全代码”升级到“参与任务流”，核心挑战转向**环境、上下文、规则和反馈**。
- Skills 的进化依赖于**可量化的验收标准**（如性能测试通过率），而非模糊的“更好”。
- 对 Gemini 2.5 Pro 的评价：Google 模型在短暂领先后又开始掉队，说明**模型竞争窗口越来越短**。

**实用工具/资源**  
- 无特定工具，但提供了工程化管理框架的思路。

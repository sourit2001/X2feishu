# AI 创作新范式：从“命名偏好”到“全景视频”，揭秘模型协同的魔法时刻

## 前言

过去几小时，AI 创作圈的核心话题已从单一的“提示词技巧”悄然转向更复杂的 **“多模型协同工作流”**。无论是 Ethan Mollick 揭示的模型“人格化”命名偏好，还是开发者们利用 GPT Image 2 与 Codex 串联打造沉浸式全景体验，都指向一个明确趋势：**理解不同AI模型的“性格”与“特长”，并将其组合成高效流水线，已成为释放创造力的关键**。这不再是简单的“哪个模型更强”的争论，而是进入了“如何让模型们各司其职、默契配合”的实战新阶段。

## 👤 博主深度速递

### Ethan Mollick：洞察模型“人格”与隐藏特性

*   **主要内容**：Ethan 分享了两项有趣且实用的观察。首先，他发现各大主流 AI 模型在生成角色名称时，存在明显的、可预测的“偏好”。例如，向 Claude 4.5 索要软件开发者，大概率会得到“Marcus Chen”；而“巫师”则多被命名为“Aldric”。其次，他指出了一个未被官方明确说明的重要事实：在使用 **GPT-imagegen-2** 时，**背后选择的语言模型（LLM）版本会显著影响图像生成质量**。更高级的“思考”或“专业”版本，在处理复杂概念时表现更优。
*   **核心观点**：
    1.  AI 模型并非完全中立，其训练数据赋予了它们独特的“文化偏好”和输出风格。了解这些偏好，有助于我们预测和引导输出。
    2.  多模态模型的性能边界，很大程度上受其核心语言模型能力的制约。为图像生成任务选择一个更强大的“大脑”（LLM），能直接提升最终成果的上限。
*   **实用工具/资源**：GPT-imagegen-2（图像生成），GPT-5.4 Thinking / Pro（作为其背后的语言模型）。
*   **来源链接**：[推文1](https://twitter.com/emollick/status/2046952100769272019), [推文2](https://twitter.com/emollick/status/2046960756608868533)

### 小虎：Codex vs. Claude Code，两种编程哲学的对决

*   **主要内容**：博主对比了 Codex 和 Claude Code 在编程任务上的不同风格。他用了一个生动的比喻：Claude Code 像是个“快男”，急于直接生成代码、解决问题；而 Codex 则更像一个“规划者”，会先构思步骤、搭建框架，再逐步实现。他认为在需要稳健和详细规划的编程场景中，Codex 的表现更可靠。
*   **核心观点**：不同的 AI 编程助手拥有截然不同的“工作性格”。**Claude Code 偏向“敏捷执行”，Codex 偏向“系统规划”**。开发者应根据任务性质（快速原型 vs. 复杂系统构建）来选择合适的工具，或让它们在不同阶段发挥作用。
*   **实用工具/资源**：Codex, Claude Code。
*   **来源链接**：[推文1](https://twitter.com/xiaohu/status/2046955773197222090)

### icreatelife：GPT Image 2 + Codex 全景创作工作流实战

*   **主要内容**：该博主是“多模型协同创作”的积极实践者，集中展示了一套利用 **GPT Image 2 生成全景图**，再通过 **Codex 将其转化为交互式或自动漫游的 3D 视图** 的完整工作流。他分享了从生成（“make equirectangular panorama of [地点]”）到编码（“make mouse-controlled 3D view” 或 “make this a smooth automatic pan”）的具体提示词。此外，他还转发了利用 GPT Image 2 生成 Photoshop 快捷键信息图等创意用例。
*   **核心观点**：
    1.  **AI 创作的未来在于“流水线”**：将擅长视觉生成的模型与擅长代码实现的模型串联，可以创造出任何单一模型无法完成的复杂、动态作品。
    2.  **提示词是连接模型的通用语言**：一套清晰的提示词，可以在不同模型间传递创作意图，实现无缝接力。
    3.  **社区共创价值巨大**：他积极鼓励分享实验成果，收集最佳用例，体现了开放协作能加速探索边界。
*   **实用工具/资源**：
    - **核心工具链**：GPT Image 2（可通过 Adobe Firefly 使用），Codex。
    - **关键提示词**：
        - 图像生成：`make equirectangular panorama of [PLACE]`
        - 代码生成：`make mouse-controlled 3D view`；`make 3d view of this panorama, make this a smooth automatic pan of the 360 view`
        - 信息图生成：`Create a clean and modern infographic showing the most essential Photoshop shortcuts, well organized and easy to read`

## 🔥 综合启发

结合以上动态，我们为开发者和 AI 创作者提炼出三条即刻可用的行动指南：

1.  **像管理团队一样管理你的 AI 工具**：不要再把 AI 模型视为单一工具。根据 **Ethan** 和 **小虎** 的观察，将它们视为拥有不同性格和专长的“团队成员”。需要创意概念和视觉时，调用“艺术家”模型（如 GPT Image 2）；需要严谨规划和代码实现时，交给“工程师”模型（如 Codex）。明确分工，才能效率最大化。

2.  **投资“提示词工程”升级为“工作流设计”**：**icreatelife** 的演示极具启发性。未来的核心竞争力，不仅在于写出单个好的提示词，更在于设计一套**能让多个模型接力完成复杂任务的提示词流程**。尝试将你的创作过程拆解为“概念-视觉化-交互/动态化”等环节，并为每个环节匹配最合适的模型和提示词。

3.  **主动测试模型的“隐藏设定”与边界**：**Ethan** 提到的“LLM 版本影响图像生成质量”是一个重要提醒。许多性能细节官方未必明说。养成习惯：对于关键任务，主动测试同一产品的不同版本或模式（如标准版、思考版、专业版），记录它们在特定任务上的输出差异。这能帮你建立自己的“模型性能图谱”，在重要时刻做出最优选择。

**本期简报核心脉络**：从模型的隐性人格，到编程的风格差异，再到跨模态的流水线协作，AI 应用正在从“工具使用”深度走向“生态协同”。理解并驾驭这种协同，将是下一波生产力爆发的关键。

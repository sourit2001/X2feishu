### 2. 归藏 (@op7418) — DeepSeek V4 深度评测：性能炸裂，但指令遵循存疑

作为国内知名的 AI 博主，归藏对 DeepSeek V4 进行了第一时间的深度测试和价格分析。

#### 主要内容
- **模型发布**：DeepSeek V4 正式发布，分为 **Pro**（总参数 1.6T，激活 49B）和 **Flash**（总参数 284B，激活 13B）两个版本，均基于 32T 数据训练。
- **价格修正**：纠正了最初的价格错误。**V4-Flash** 百万输入仅 ¥0.2，输出 ¥2；**V4-Pro** 输入 ¥1，输出 ¥24。性价比极高，尤其是 Flash 版本。
- **功能支持**：支持 JSON 输出、工具调用、对话前缀续写、FIM 补全等，功能全面。
- **实测翻车**：归藏测试后发现，**DeepSeek V4 无法正常调用 Claude Skill**，指令遵循和工具调用效果很差，甚至无法读取模板，直接“自己实现了一个网页”。这暴露了其在复杂 Agent 任务上的短板。

#### 核心观点
DeepSeek V4 在**价格和基础能力**上极具竞争力（尤其是 Flash 版本），堪称“性价比之王”。但在**指令遵循和工具调用**等高级 Agent 能力上，与 Claude 和 GPT 仍有明显差距。开发者需谨慎评估其在复杂工作流中的表现。

### 3. 宝玉 (@dotey) — DeepSeek V4 技术细节全解析：百万上下文与 Agent 优化

宝玉的总结非常专业，侧重于 DeepSeek V4 的技术创新和实际应用场景。

#### 主要内容
- **百万上下文成标配**：DeepSeek V4 全系列（不分版本）均支持 1M 上下文，这已超越许多竞品。
- **结构创新**：采用全新的 **DSA 稀疏注意力机制**，在 token 维度进行压缩，大幅降低长上下文场景下的计算和显存需求。
- **Agent 专项优化**：V4 专门针对 Claude Code、OpenClaw、CodeBuddy 等主流 Agent 产品进行了适配优化，并展示了其生成的 PPT 内页示例。
- **API 兼容**：API 支持 OpenAI ChatCompletions 和 Anthropic 接口，迁移成本极低。Base URL 不变，仅需修改 model 参数为 `deepseek-v4-pro` 或 `deepseek-v4-flash`。

#### 核心观点
DeepSeek V4 的战略非常清晰：**用开源和极致的性价比，抢占 Agent 和长上下文应用的市场**。其技术路径（DSA 注意力）和生态兼容策略（兼容 OpenAI/Anthropic 接口）都旨在降低开发者门槛。

### 4. Ethan Mollick (@emollick) — 模型能力横向对比：GPT-5.5 与 DeepSeek V4 的“进化”之战

作为沃顿商学院的教授，Ethan 的测试更具学术性和趣味性。

#### 主要内容
- **GPT-5.5 的“进化”能力**：Ethan 让多个模型“用一句 Prompt 构建一个从公元前 3000 年到公元 3000 年港口小镇演化的 3D 模拟”。只有 GPT-5.5 真正理解了“进化”二字，不仅替换建筑，还改变了环境设定。
- **DeepSeek V4 初体验**：Ethan 测试了 DeepSeek V4 Pro（专家模式），并生成了他的第一个“TiKZ Sparks 独角兽”图片，效果不错。
- **横向对比**：Ethan 将 DeepSeek V4 Pro 的 3D 模拟结果与 Kimi K2.6 对比，直言“不知道差距为什么这么大”，暗示 DeepSeek V4 在复杂创意任务上仍有差距。

#### 核心观点
Ethan 的测试揭示了当前模型的**能力分层**：GPT-5.5 在**理解复杂、抽象指令**（如“进化”）上表现突出；DeepSeek V4 在**执行具体、结构化任务**（如生成 TiKZ 图片）上表现良好。模型之间的“智商”鸿沟依然存在。

## 🔥 综合启发

1.  **Agent 应用的“性价比”时代已经到来**：DeepSeek V4 Flash 的极低定价，将彻底改变 Agent 应用的成本结构。对于开发者而言，**从“用不起”到“随便用”**，意味着可以大规模部署 AI 代理来完成文档处理、代码审查、数据抓取等任务。建议立即测试 DeepSeek V4 Flash 在自家 Agent 框架中的表现。

2.  **长上下文不再是“奢侈品”，而是“标配”**：DeepSeek V4 将百万上下文作为全系标配，GPT-5.5 也支持视频输入。这意味着，**处理整本书、整个代码库、甚至一段长视频**将成为可能。建议开发者开始思考如何利用超长上下文，设计全新的、无需分段处理的 AI 应用。

3.  **Open Source 与 Closed Source 的博弈进入新阶段**：DeepSeek V4 的全面开源，直接挑战了 GPT-5.5 和 Claude 的闭源生态。开源模型在**成本、可控性和定制化**上优势巨大，但闭源模型在**指令遵循、复杂推理和创意生成**上依然领先。**“混合架构”**（核心任务用闭源模型，批量任务用开源模型）可能成为未来企业的最佳实践。

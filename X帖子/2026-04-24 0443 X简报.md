## 👤 博主深度速递

### 1. Sam Altman (@sama) — OpenAI CEO

**主要内容：**
Sam Altman 在凌晨连续发布多条推文，正式官宣 **GPT-5.5** 的发布。他强调，新模型不仅“更聪明”，而且“更快”。在推理速度上，它保持了与 GPT-5.4 相同的每 token 延迟，但完成相同任务所需的 token 数更少。这意味着用户将获得“又快又省”的体验。同时，他宣布 GPT-5.5 已同步上线 **Codex**，并预告了 Codex 即将迎来一大波新功能。

**核心观点：**
1.  **迭代部署是安全策略：** Altman 明确表示，OpenAI 信奉“迭代部署”。虽然 GPT-5.5 已经很强，但他们期待快速改进。这是他们安全策略的一部分，旨在让世界更好地适应 AI 的韧性。
2.  **OpenAI 正在成为一家“AI 推理公司”：** 他特别表扬了推理团队，并指出“在很大程度上，我们现在必须成为一家AI推理公司。” 这暗示了模型架构和服务优化的重大转向。
3.  **Codex 的企业级部署：** 他提到与 NVIDIA 合作，在公司内部推广 Codex，并开放给其他企业申请。

**实用工具/资源：**
- **GPT-5.5**：已在 ChatGPT (Plus/Pro/Business/Enterprise) 和 Codex 中可用。
- **Codex**：与 GPT-5.5 深度集成，即将推出大量新功能。

### 2. 宝玉 (@dotey) — 知名 AI 博主 & 开发者

**主要内容：**
宝玉老师从开发者视角，对 GPT-5.5 和 Codex 进行了深度解读。他总结了 GPT-5.5 的“核心卖点”：**更聪明但不变慢**。同时，他敏锐地指出了 Claude Code 和 Codex 在使用“Skills”时的关键区别：执行模型不同（Claude vs. GPT），以及运行环境（Harness）不同。他还展示了 GPT-5.5 在“写作”上的巨大进步，用“寓言”的方式解释一个抽象概念，效果惊人。

**核心观点：**
1.  **Codex 的进化方向是“智能体”：** 他详细分析了 Codex 一口气推出的五大能力升级，核心是从“写代码的工具”变成“帮你干活的智能体”，其中最大的亮点是**浏览器操控**。
2.  **模型“性格”差异影响 Skills：** 不同模型（Claude vs. GPT）对提示词的偏好和能力不同，这导致同一个 Skills 在不同 Agent 中的表现会天差地别。
3.  **写作能力显著提升：** 他通过一个巧妙的 Prompt，让 GPT-5.5 写出了非常“说人话”的寓言故事，认为其文本生成的自然度有了质的飞跃。

**实用工具/资源：**
- **GPT-5.5**：已在 ChatGPT 和 Codex 中可用。
- **Codex 新能力**：浏览器控制、Sheets & Slides、Docs & PDFs、OS 级听写、自动审查模式。
- **Prompt 示例**：通过写寓言解释“Agent Harness”概念。

## 🔥 综合启发

结合以上所有动态，我们可以提炼出以下几条对开发者和 AI 使用者的具体建议：

1.  **拥抱“智能体”时代，重构工作流：** GPT-5.5 和 Codex 的进化，清晰地指向了“智能体”的未来。不要再把 AI 仅仅当作一个问答机器人或代码补全工具。尝试使用 Codex 这样的平台，让它去**自主完成一个完整任务**（如“研究竞品并生成一份 PPT”）。你将成为 AI 的“管理者”而非“操作员”。

2.  **善用“组合拳”，而非“站队”：** 正如 Riley Brown 和宝玉老师所展示的，不同模型和工具有各自的强项。GPT-5.5 在代码生成和复杂推理上很强，Claude Code 在 UI 设计上可能有优势。聪明的做法是**在一个平台（如 Codex）内，灵活调用不同模型**，取长补短，实现 1+1 > 2 的效果。

3.  **关注“迭代部署”带来的机遇与风险：** Sam Altman 的“迭代部署”策略，意味着模型能力会快速、持续地提升。这对开发者是巨大的机遇，可以第一时间利用最新能力。但同时也意味着**你需要不断学习和适应**。不要固守于某个版本的“最佳实践”，保持对模型能力边界的敏锐感知，并做好应对“锯齿状”不完美能力的准备。

### 2. God of Prompt (@godofprompt)

如果说马斯克在谈“道”，那 God of Prompt 则是在拆解“术”。他今天分享的内容，堪称是对 Google Gemini API 一次重大、且极其实用的技术拆解。

#### 主要内容

God of Prompt 指出，Google 对 Gemini API 进行了一次根本性的结构升级。老版的 Gemini API 像一个“黑箱”自动售货机（vending machine），你输入提示词，它直接输出答案，中间发生了什么完全不可见。

**重大更新来了：** 新版的 Gemini API 现在为开发者提供了一个清晰的、结构化的“时间线”（structured timeline）。它会把模型在生成最终答案前，每一个步骤都清晰地拆解和标记出来。这包括：
-   **思考 (Thinking)**
-   **搜索 (Searching)**
-   **工具调用 (Tool calls)**
-   **最终输出 (Final output)**

这意味着，开发者现在可以像调试普通代码一样，去“观察”和“引导”AI模型的推理过程，而不仅仅是等待最终结果。

#### 核心观点

-   **AI开发的“可调试性”时代来临：** God of Prompt 强调了这次更新的里程碑意义。当模型的思考过程被透明化，开发者就能精准定位问题所在。比如，是因为模型思考逻辑错了？还是外部工具调用失败了？这让优化和调试AI应用变得前所未有的高效和精准。
-   **从“黑箱”到“白盒”：** 这是对AI模型可解释性（Explainability）的重大胜利。对于那些需要高可靠性、安全性的应用场景（如金融、医疗、自动驾驶），能够审查AI的“思考过程”是极其关键的一步。

#### 实用工具/资源

-   **工具/资源：** Google Gemini API 最新版（支持结构化推理轨迹）。
-   **核心概念：** 结构化时间线 (Structured Timeline)、模型可解释性 (Model Explainability)。

## 🔥 综合启发

结合这两位大佬的输出，我们可以得到以下几点非常具有现实意义的启发：

1.  **“制造”的思维，不仅限于物理世界，更在数字世界：** 马斯克强调在现实世界通过物理制造创造价值；而 God of Prompt 介绍的更新，则是在数字世界给了开发者“制造”好AI应用的工具。**行动建议：** 如今的“制造者”，不仅要会造车，更要会“造”AI应用。学习如何利用新API的结构化能力，去构建更可靠、更可控的AI产品，是成为新一代“Maker”的关键。

2.  **理解“过程”比得到“结果”更重要：** 马斯克对宇宙的好奇心，是对“过程”与“本质”的追求；Gemini API的更新，让开发者可以查看AI的“思考过程”。**行动建议：** 无论是学习新技术还是调试AI模型，别再满足于“它对了/它错了”这样的二元结果。要像程序员Debug一样，深入分析AI的推理链。这不仅能提升你的技术能力，更能培养你在这个不确定时代里，进行批判性思考和深入探究的习惯。

3.  **对齐你的哲学与技术：** 马斯克的核心哲学（好奇、制造）驱动了他的技术决策（造火箭、造电车）；而AI的透明化更新，本质上也是一次技术对“人本主义”的讨好——让技术更可解释、更可信、更可控。**行动建议：** 作为开发者或科技爱好者，你的“哲学”是什么？你追求的是效率、是开放、还是可靠性？下次选择技术栈或设计产品时，先问问自己，这项技术是否与你的底层价值观和核心目标对齐。只有当“道”与“术”统一时，你才能创造出真正有影响力的东西。
